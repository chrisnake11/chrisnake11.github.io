# 进程通信

## 共享内存

两个进程的虚拟内存映射到同一块物理内存上。系统调用mmap()

```c++
key_t ftok(const char* path, int proj_id) 将本地路径转化为一个系统进程通信的key
void* shmget(key_t key, size_t size, int shmflg) 获取共享内存
void* shmat(int shmid, const void * shmaddr, int shmflg);
int shmdt(const void * shmaddr)
int shmctl(int shmid, int cmd, struct shmid_ds* buf); 控制共享内存段。
```


## 信号量

实现多个进程访问共享资源之间的互斥与同步。并不用于缓存进程之间的通信数据。


## 消息队列

存在于内核中，生命周期于操作系统内核相同。

缺点：能够内核分配给消息链表的内存空间有限；消息队列无法及时响应，因为每次读取和写入都要从内核中拷贝消息链表中的数据到用户空间。


## 管道

创建一个管道，一端读取，一端写入。

1. 匿名管道：只存在于内存当中，通信的数据无格式且大小受限，且通信的方向是单项的。适用于父子进程之间的通信。

2. 命名管道：在文件系统中创建一个类型pipe的设备文件，两个毫无关系的进程就会通过这个设备文件进行通信。

> 不管是匿名管道还是命名管道，二者的写入数据都是缓存在内核中的，读取数据也是从内核的缓存中读取。

## 信号

由操作系统提供的，用于异常状态下的进程之间的异步的通信方式。某一进程可以任意时刻发送信号给指定进程，进程收到信号后进行相应。

进程可以设置SIGIGN来忽略捕获到的信号。


## socket()

用于不同机器上的进程的通信。
